# 💸 Telegram Expense Tracker Bot

An AI-assisted, multi-user **Telegram bot** for recording, managing and analysing
personal expenses through a conversational interface. Built with a clean,
layered architecture so features (OCR, bank imports, extra platforms) can be
added with minimal churn.

---

## ✨ Features

| Area | What it does |
|------|--------------|
| **Implicit auth** | Every Telegram `user_id` is a unique account; all data is isolated per user. |
| **Guided entry (FSM)** | Step-by-step add-expense flow: amount → category → merchant → date → payment → notes → confirmation. |
| **Natural language** | Type `Spent ₹450 at Starbucks today` — the bot extracts amount, currency, merchant, category and date, and only asks for what's missing. |
| **Global currencies** | `250`, `₹450`, `€30`, `35 USD`, `1,250.50` — symbols and ISO codes both supported. |
| **Reports** | Today, Weekly and Monthly reports with category/merchant breakdowns, budget utilisation, largest expense and daily averages. |
| **Budgets** | Per-category monthly limits with automatic over-budget alerts during entry. |
| **Statistics** | Top/lowest/most-frequent categories, averages, weekly & monthly totals, top merchants. |
| **Exports** | CSV (built-in), Excel & PDF (optional `analytics` extra / extension hook). |
| **Reminders** | Configurable per-user daily reminder ("Did you spend anything today?"), timezone-aware. |
| **Settings** | Currency, timezone, default payment method, export format and reminder time. |

## 🏗 Architecture

Clean, layered separation — dependencies point inward (presentation → services → domain → persistence):

```
expense_bot/
├── core/           # Domain: config, schemas, categories, currency, NLP parser, logging
├── db/             # Persistence: SQLAlchemy models, async session, repositories
├── services/       # Application/business logic (user, expense, budget, report, stats, export)
├── bot/            # Presentation: Telegram handlers, keyboards, FSM states, formatting
│   ├── handlers/   #   menu, add_expense, reports, statistics, budget, settings, nlp
│   ├── reminders.py
│   └── application.py   # wires the Application + handler registration order
└── main.py         # entrypoint (long polling)
alembic/            # database migrations
tests/              # unit (parsing) + integration (services) tests
```

**Tech stack:** Python 3.12 · python-telegram-bot v22 · SQLAlchemy 2 (async) ·
Pydantic v2 · Alembic · APScheduler (via PTB JobQueue) · PostgreSQL (prod) /
SQLite (dev) · Docker.

## 🚀 Quick start (local, SQLite)

### 1. Prerequisites
- Python **3.12+**
- A Telegram bot token from [@BotFather](https://t.me/BotFather)

### 2. Get a bot token
1. Open Telegram, message **@BotFather**, send `/newbot`.
2. Choose a name and username; BotFather returns a token like `123456:ABC-DEF…`.

### 3. Install
```bash
git clone <your-repo-url> expense-tracker
cd expense-tracker

python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -e ".[dev]"            # add ,analytics for Excel/PDF export
```

### 4. Configure
```bash
cp .env.example .env
# edit .env and set BOT_TOKEN=<your token from BotFather>
```

Key variables (see [.env.example](.env.example)):

| Variable | Default | Notes |
|----------|---------|-------|
| `BOT_TOKEN` | — | **required** |
| `DATABASE_URL` | `sqlite+aiosqlite:///./expense_bot.db` | swap for PostgreSQL in prod |
| `ENVIRONMENT` | `development` | `production` disables the SQLite auto-create-tables convenience |
| `LOG_LEVEL` | `INFO` | |
| `DEFAULT_CURRENCY` | `USD` | new users' default |
| `DEFAULT_TIMEZONE` | `UTC` | new users' default |
| `REMINDERS_ENABLED` | `true` | |

### 5. Run
```bash
python -m expense_bot.main
```
On SQLite the tables are created automatically. Open Telegram, message your bot,
and send `/start`.

## 🐘 Production (PostgreSQL + Docker)

```bash
export BOT_TOKEN=<your token>
docker compose up --build
```

`docker compose` starts PostgreSQL, waits for it to be healthy, runs
`alembic upgrade head`, then launches the bot. Data persists in the `pgdata`
volume.

### Running migrations manually
```bash
# create a new autogenerated migration after changing models
alembic revision --autogenerate -m "describe change"

# apply migrations
alembic upgrade head
```
> The database URL is read from `DATABASE_URL` (via `alembic/env.py`), so it is
> never stored in `alembic.ini`.

## 💬 Using the bot

- `/start` — main menu.
- **➕ Add Expense** — guided flow with inline buttons.
- **Free text** — e.g. `lunch 250 today`, `Spent €30 at Uber yesterday`.
- **📊 Today / 📅 Monthly / 📈 Statistics** — reports & insights.
- **💰 Budget** — set/remove monthly category limits.
- **⚙️ Settings** — currency, timezone, payment, export format, reminder time.
- `/cancel` — abort the current step; `/help` — quick reference.

## 🔌 Extension points

The layering keeps future work localised:

- **AI/NLP extraction** — replace the rule-based `core/parsing.parse_expense_text`
  with an LLM call (OpenRouter / Ollama). Callers are unchanged.
- **PDF/Excel export** — implement `services/export_service.to_pdf_bytes`
  (hook already wired into the export handler with graceful fallback).
- **Charts** — add Matplotlib/Plotly rendering in a new service and attach the
  image in the report handlers.
- **Webhook mode** — swap `run_polling` in `main.py` for `run_webhook` behind
  FastAPI/Nginx; no changes needed elsewhere.
- **New platforms** — reuse `services/` from a different presentation layer.
